# 📅 Git日报生成技能

欢迎使用 Git日报生成技能！这个技能能够智能分析你的Git提交日志，生成简洁明了的工作日报，帮助你快速了解开发进展和关键工作成果。

## 快速开始

### 安装技能

#### 全局安装（推荐）
```bash
cp -r templates/.claude/skills/daily-report ~/.claude/skills/
```

#### 项目级别安装
```bash
mkdir -p .claude/skills
cp -r templates/.claude/skills/daily-report .claude/skills/
```

### 基本使用

#### 生成昨天的日报
```
用户：生成昨天的日报
助手：我将分析昨天的Git提交日志并生成日报...
```

#### 生成指定日期的日报
```
用户：生成2025-10-30的日报
助手：我将分析2025年10月30日的Git提交日志...
```

## 功能特性

### 🎯 智能分析
- **自动识别**：智能识别功能开发、问题修复、代码优化等不同类型的工作
- **重要性评估**：自动评估提交的重要性和影响范围
- **数据量化**：统计代码变更、问题修复等量化指标

### 📊 简洁输出
- **3-4行描述**：每行聚焦一个主要工作项
- **专业语言**：使用清晰、专业的表达方式
- **数据支撑**：附带代码统计、问题修复数量等量化数据

### 🔧 灵活配置
- **日期范围**：支持指定任意日期或日期范围
- **输出格式**：提供标准和简洁两种输出格式
- **内容筛选**：可指定关注的工作类型

## 输出格式说明

### 标准格式
- **标题**：包含日期标识
- **工作项**：每行一个主要工作类型
- **数据嵌入**：自然包含代码统计、问题修复数量等量化指标
- **专业描述**：清晰说明工作内容和成果

### 简洁格式
- **紧凑布局**：使用符号和缩略表达
- **关键信息**：保留核心数据和量化指标
- **快速阅读**：便于快速浏览和分享

## 技能工作原理

### 1. 数据获取
- 使用Git命令获取指定日期的提交日志
- 提取提交哈希、作者、日期、提交信息等数据
- 按时间倒序排列提交列表

### 2. 智能分析
- 识别提交类型（功能、修复、优化、文档等）
- 评估提交的重要性和影响范围
- 提取提交信息中的核心内容

### 3. 日报生成
- 按重要性排序工作内容
- 生成3-4行简洁描述
- 嵌入量化数据指标
- 优化输出格式和表达

## 配置选项

### 日期范围
- **默认**：前一天（当前日期减1天）
- **指定日期**：任何有效的YYYY-MM-DD格式
- **日期范围**：开始日期到结束日期

### 输出格式
- **标准格式**：详细的段落描述（默认）
- **简洁格式**：紧凑的列表形式

### 内容筛选
- **全部类型**：包含所有工作类型
- **特定类型**：只关注指定类型的工作

## 故障排除

### 技能没有被激活
- **问题**：发出请求后，Claude没有使用daily-report技能
- **解决方案**：
  - 确认技能已正确安装
  - 使用更明确的请求，如"使用daily-report技能生成日报"
  - 检查技能描述是否匹配你的请求

### 没有找到提交记录
- **问题**：技能运行但提示没有找到提交记录
- **解决方案**：
  - 确认指定的日期范围内有提交
  - 检查Git仓库状态和提交历史
  - 验证日期格式是否正确

### 输出不符合预期
- **问题**：日报内容不够准确或完整
- **解决方案**：
  - 提供更具体的需求描述
  - 指定关注的工作类型
  - 要求特定的输出格式
